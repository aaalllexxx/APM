# AEngine Package Manager (APM) v2.0

`apm` — мощная утилита командной строки для конфигурации, создания и управления проектами и модулями в экосистеме AEngine.

### 🌟 Новое в v2.0
- **Портативный режим (Portable)**: Конфигурация теперь хранится локально в директории APM. Никаких зависимостей от `%APPDATA%` или профиля пользователя.
- **Subcommand System**: Динамическая загрузка подкоманд из установленных модулей (например, `apm sec sign`).

---

## 🚀 Установка и удаление

**Установка:**
После клонирования репозитория перейдите в папку `scripts` и запустите от имени администратора:
```cmd
setup.bat
```

**Удаление:**
Для удаления менеджера из системы:
```cmd
delete.bat
```

## 🛠️ Управление проектами

### `apm create`
Создаёт новый проект AEngineApps с нуля. Генерирует файловую структуру, `config.json` и стартовый код `main.py`.
Автоматически регистрирует проект в системеAPM.

### `apm init`
Регистрирует **существующую** директорию как проект AEngineApps (создаёт `.apm` и добавляет в глобальный реестр APM).

### `apm delete`
Полностью удаляет проект с диска и убирает его из реестра APM. Запрашивает подтверждение перед удалением.

### `apm run`
Запускает скрипт `main.py` текущего проекта. Если проект запущен в режиме webview, открывает окно.

### `apm config`
Интерактивный конфигуратор файла `config.json`. Позволяет настроить:
- `debug` (true/false)
- `view` (app/web)
- `screen_path` (папка с экранами)
- `routers` (auto или ручная настройка)
- Файлы для исключения из сборки.

### `apm select`
Позволяет выбрать проект по умолчанию для глобального взаимодействия.

### `apm upgrade`
Обновляет и интегрирует последнюю версию фреймворка `AEngineApps` в текущий проект. Команда удаляет старую папку AEngineApps и скачивает самую свежую версию из репозитория, не затрагивая конфигурацию и файлы самого проекта.

## 🧭 Навигация

### `apm list`
Выводит красивую таблицу всех зарегистрированных проектов. Также проверяет их статус существования на диске:
- `[✓]` — папка проекта существует
- `[✗]` — проект был удалён или перемещён

### `apm goto`
Позволяет быстро перейти в консоли в папку любого из зарегистрированных проектов с помощью интерактивного выбора стрелочками.
*Примечание: работает, потому что `apm` вызывает `apm.bat`, который поддерживает смену директории `cd`.*

### `apm unregister`
Удаляет проект из глобального реестра APM (полезно для проектов со статусом `[✗]`), **не удаляя** при этом сами файлы проекта с диска.

## 🧩 Управление модулями

### `apm install <git_url>`
Клонирует и устанавливает модуль (библиотеку) в текущий проект. Можно использовать флаг `-g` для глобальной установки во все проекты:
```bash
apm install https://github.com/user/module
apm install -g https://github.com/user/module_global
```

### `apm remove`
Интерактивно (стрелками) позволяет выбрать и удалить установленный модуль из проекта.

### `apm modules`
Выводит список всех установленных модулей в текущем проекте.

### `apm develop`
Генератор шаблонов для разработчиков. 
Поддерживает две команды:
1. `apm develop module` — генерирует болванку пакета APM для расширения функционала консоли
2. `apm develop screen` — генерирует класс-наследник `Screen` и HTML-шаблон для него

## ⚙️ Прочее

### `apm update`
Синхронизирует репозиторий APM с GitHub (`git pull`), обновляя менеджер проектов до последней версии.

### `apm docs`
Открывает оффлайн-документацию (если есть) или направляет на Github.

### `apm help` (или `apm --help`)
Выводит полную категоризированную справку по всем командам менеджера.
